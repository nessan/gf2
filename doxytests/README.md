# README

This directory contains lots of test files with names like `test_Foo.cpp`.

The files were generated by running the Python script `cmake/doxytest.py` on the corresponding header file `Foo.h`.

There is a corresponding `CMake` script `doxytest.cmake` that automatically invokes `doxytest.py` whenever the header file is updated.

The overall effect is similar to the builtin documentation test facility in Rust.

In `Foo.h` we have a header comment for each method looking something like:

````cpp
/// @brief A brief description of the method in question.
///
/// Perhaps more detailed notes, caveats, etc. followed by
///
/// # Examples
/// ```
///     example code written in the form of a test ....
/// ```
````

The Python script extracts all those examples, putting each one in its own block with some numbering information.
When you build and run `test_Foo` all the test blocks are executed.

If any test fails you get information about the failure and also can see the line in the header file where the particular example block came from.

_Note: There is no point editing `test_Foo.cpp` by hand as it will be overwritten by the script on the next run._
